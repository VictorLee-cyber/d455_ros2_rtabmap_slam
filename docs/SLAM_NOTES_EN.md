# D455 RGB-D SLAM Notes

This document explains the basic principles, software dependencies, ROS2 topic structure, RTAB-Map parameters, and common troubleshooting steps of this project.

This project is based on Intel RealSense D455, ROS2 Humble, and RTAB-Map. It provides a reproducible RGB-D SLAM demo. The goal is not to implement a SLAM algorithm from scratch, but to build an engineering workflow for D455 camera setup, RGB-D data synchronization, RTAB-Map configuration, map saving, and practical troubleshooting.

---

## 1. Project Pipeline

```text
Intel RealSense D455
        ↓
realsense2_camera
        ↓
RGB Image / Aligned Depth Image / CameraInfo
        ↓
RTAB-Map RGB-D SLAM
        ↓
Visual Odometry / 3D Point Cloud Map / 2D Occupancy Grid
        ↓
RTAB-Map Database / Nav2 Map Export
```

The D455 provides RGB and depth images. The `realsense2_camera` package publishes camera data into ROS2. RTAB-Map estimates camera motion and builds a map from RGB-D data. The 2D occupancy grid can later be exported for Nav2 navigation.

---

## 2. Why RTAB-Map Instead of FAST-LIO

FAST-LIO / FAST-LIO2 is mainly designed for LiDAR-Inertial Odometry, which uses LiDAR point clouds and IMU data.

D455 is an RGB-D camera. Although it can publish `PointCloud2`, the point cloud is generated by projecting the depth image, not by a real LiDAR scan. Therefore, this project uses RTAB-Map for RGB-D SLAM.

Recommended combinations:

```text
D455 + RTAB-Map      → RGB-D SLAM
Mid360 + FAST-LIO2   → LiDAR SLAM
D455 + Nav2          → Near-field obstacle avoidance and visual navigation support
```

## 4. RealSense D455 ROS2 Topics

After launching D455, common topics include:

```text
/camera/camera/color/image_raw
/camera/camera/color/camera_info
/camera/camera/aligned_depth_to_color/image_raw
/camera/camera/depth/image_rect_raw
/camera/camera/extrinsics/depth_to_color
```

RTAB-Map mainly uses:

```text
RGB image:
/camera/camera/color/image_raw

Aligned depth image:
/camera/camera/aligned_depth_to_color/image_raw

Camera intrinsics:
/camera/camera/color/camera_info
```

Check topic frequency:

```bash
ros2 topic hz /camera/camera/color/image_raw
ros2 topic hz /camera/camera/aligned_depth_to_color/image_raw
```

Check frame id:

```bash
ros2 topic echo /camera/camera/color/camera_info --once | grep frame_id
```

This project uses the following RTAB-Map frame:

```text
frame_id:=camera_link
```

This is more suitable for ground-robot style mapping and avoids confusion caused by directly using the optical frame.

## 6. RTAB-Map Launch and Parameters

Launch SLAM:

```bash
ros2 launch d455_rtabmap_demo rtabmap_d455.launch.py
```

Important parameters:

```text
frame_id:=camera_link
```

Uses the camera body frame as the reference frame.

```text
approx_sync:=true
```

RGB and depth images may not have exactly the same timestamp. Approximate synchronization improves robustness.

```text
visual_odometry:=true
```

Enables RTAB-Map visual odometry.

```text
qos:=2
```

Matches the sensor-data QoS commonly used by RealSense image topics.

```text
delete_db_on_start:=true
```

Creates a new database each time. Set it to `false` to continue using an existing map.

```text
--Reg/Force3DoF true
```

Constrains motion to ground-robot planar motion and reduces map tilt caused by roll / pitch drift.

```text
--Rtabmap/DetectionRate 2
```

Limits RTAB-Map processing rate to reduce CPU load.

```text
--Vis/MaxFeatures 800
--Kp/MaxFeatures 800
```

Limits visual features and keypoints to balance tracking quality and runtime performance.

## 7. USB3 Check

D455 should run in USB3 mode.

Check USB type:

```bash
rs-enumerate-devices | grep "Usb Type"
```

Expected result:

```text
Usb Type Descriptor : 3.x
```

Check USB tree:

```bash
lsusb -t
```

Expected result:

```text
Driver=xhci_hcd, 5000M
```

Abnormal result:

```text
Usb Type Descriptor : 2.1
Driver=ehci-pci, 480M
```

This means D455 is running in USB2 mode, which may cause low FPS, dropped frames, or device disconnection.

Project script:

```bash
ros2 run d455_rtabmap_demo check_usb3.sh
```

Common solutions:

```text
Use a real USB3 data cable
Connect directly to a USB3 port
Avoid USB hubs and docking stations
Enable USB 3.1 controller in VMware
Reconnect D455 to the virtual machine
```

```bash

```

The database stores:

```text

Camera trajectory
RGB images
Depth images
Visual features
Graph structure
Loop closure information
```

Save current database:

```bash
ros2 run d455_rtabmap_demo save_rtabmap_db.sh
```

Open default database:

```bash
ros2 run d455_rtabmap_demo open_rtabmap_db.sh
```


## 9. Export 2D Map for Nav2

RTAB-Map can publish a 2D occupancy grid on `/map`.

Save the map:

```bash
ros2 run d455_rtabmap_demo save_2d_map.sh
```

The output files are usually:

```text

```

These files can be used later by Nav2.

## 10. Common Issues

### 10.1 Package 'realsense2_camera' not found

Reason: RealSense ROS2 wrapper is not installed.

Solution:

```bash
sudo apt install -y ros-humble-realsense2-*
```

### 10.2 xioctl(VIDIOC_QBUF) failed, No such device

Reason: The camera disconnects during streaming, commonly caused by USB2 mode, unstable cable, or insufficient bandwidth.

Solutions:

```text
Use USB3 mode
Change to a USB3 data cable
Avoid USB hubs
Lower resolution
Disable pointcloud and IMU
```

### 10.3 Map is tilted

Reason: The optical frame is used incorrectly or 6DoF visual odometry estimates roll / pitch drift.

Solutions:

```text
Use frame_id:=camera_link
Enable --Reg/Force3DoF true
Move the camera smoothly
Avoid dynamic objects
```

### 10.4 RTAB-Map is slow

Reasons:

```text
High resolution
Too many visual features
RViz and rtabmap_viz running at the same time
Heavy 3D point cloud visualization
VMware overhead
```

Solutions:

```text
Use fast mode
Disable RViz
Reduce DetectionRate
Reduce MaxFeatures
Avoid long-term dense 3D map visualization
```

### 10.5 The camera must move very slowly

Reason: Visual SLAM relies on feature matching between adjacent frames. Fast motion reduces overlap between frames and may cause tracking failure.

Solutions:

```text
Use 30Hz fast mode
Move smoothly
Avoid quick rotation
Keep enough texture in the image
Avoid white walls, screens, reflections, and moving people
```
